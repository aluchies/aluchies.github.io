Curse of dimensionality

Last week in the machine learning class that I am taking, the instructor was discussing the curse of dimensionality (i.e., how strange things can happen when working with high dimensional data). One of the examples he gave is simple to tryout, so I implemented it in an IPython Notebook.

import numpy as np
import matplotlib as mpl
import matplotlib.pyplot as plt

The idea is to generate many n-dimensional vectors with coordinates that are i.i.d. with standard normal distribution. Intuitively, we expect to see most of the data centered at the origin because the expected value of these random vectors is zero. The figure below confirms this intution for the one-dimensional case.

In [3]: 
ndim = 1
nsim = 10 ** 5
norm_rv = norm(0, 1)
data = norm_rv.rvs((nsim, ndim))

fig = plt.figure(figsize=(4,3))
ax = plt.subplot(1,1,1)
n, bins, patches = ax.hist(data, bins=25, normed=True, color=pastel1.mpl_colors[0], edgecolor=pastel1.mpl_colors[0])
ax.set_ylim(0., 0.5)
ax.set_xlim(-6., 6.);

The distances between the generated data points and the origin are displayed in the figure below. As expected, the distribution of these distances is concentrated near zero.

Interesting things start to happen in higher dimesions. Below are distance distributions displayed as a function of the number of dimensions.

In [5]: 
fig = plt.figure(figsize=(8, 12))

dim_range = np.arange(12) + 1
for ndim in dim_range:
    data = norm_rv.rvs((nsim, ndim))
    dist = np.sqrt(np.sum(data ** 2, axis=1))
    
    ax = plt.subplot(5,3,ndim)
    n, bins, patches = ax.hist(dist, bins=25, normed=True, color=pastel1.mpl_colors[1], edgecolor=pastel1.mpl_colors[1])
    ax.set_ylim(0, 1)
    ax.set_xlim(0, 10)
    ax.text(2.3, 0.85, '# of dims: ' + str(ndim), fontsize=14)
    ax.text(2.5, 0.73, 'mean: ' + str(round(dist.mean(), 3)), fontsize=14)
    if ndim >= 13 or ndim <= 3:
        ax.set_xlabel('distance from the origin', fontsize=14)
    
fig.tight_layout();

As the number of dimensions increases, the data is no longer concentrated near the origin. Instead it clusters near an n-dimensional spherical shell centered at the origin and having a radius that depends on the the number of dimensions as illustrated in the figure below.

In [6]: 
mean_dist = []
dim_range = np.arange(50) + 1
for ndim in dim_range:
    data = norm_rv.rvs((nsim, ndim))
    dist = np.sqrt(np.sum(data ** 2, axis=1))
    mean_dist.append(dist.mean())

fig = plt.figure(figsize=(4,3))
ax = plt.subplot(1,1,1)
ax.plot(dim_range, mean_dist, '-', color=pastel1.mpl_colors[0], label='mean distance')
ax.plot(dim_range, np.sqrt(dim_range - 1), '--', color=pastel1.mpl_colors[1], label=r'$\sqrt{n}$')
ax.set_xlabel('# of dims')
ax.set_xlim(1,50)
ax.set_ylim(0, 11)
ax.legend(loc=2);

The example illustrates the need to be careful when handling random data in higher dimensions because it's easy to assume things about your data (e.g., being concentrated at the origin) that may be true for one-, two-, or three-dimensions (dimensionalities that the human mind can understand easily), but not true in higher dimensions.
